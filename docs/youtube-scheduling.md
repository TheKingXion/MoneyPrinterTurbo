# YouTube scheduling

MoneyPrinterTurbo supports two YouTube publication scheduling modes.

## Continuous interval

Videos are scheduled continuously from the selected start date and time using
`schedule_interval_minutes`. For example, a 30 minute interval starting at
21:00 produces 21:00, 21:30, 22:00, and 22:30.

## Daily blocks

Videos are distributed across consecutive days. `schedule_videos_per_day`
controls each day's capacity and `schedule_interval_minutes` controls the time
between videos inside each daily block.

For 16 videos, 4 videos per day, starting at 21:00:

```text
14/07: 21:00, 21:15, 21:30, 21:45
15/07: 21:00, 21:15, 21:30, 21:45
16/07: 21:00, 21:15, 21:30, 21:45
17/07: 21:00, 21:15, 21:30, 21:45
```

The final partial day keeps the same cadence. Existing future publications in
`storage/youtube_upload_log.json` are skipped when allocating new slots.

## Timezone

`schedule_timezone = "local"` uses the computer's local IANA timezone. The Web
UI displays local dates and exact times. MoneyPrinterTurbo converts them to UTC
RFC3339 values before sending `publishAt` to YouTube.

Scheduled YouTube videos are uploaded as private because YouTube requires a
private video when `publishAt` is supplied.

`upload_interval_minutes` controls the delay between API uploads. It is separate
from the publication schedule.

## Batch generation

The Web UI batch generator is organized as a preparation flow: ideas, shared
video settings, schedule, and final review. Ideas can be pasted manually or
generated by the configured LLM as realistic, inspiring story briefs. Generated
ideas and optional YouTube titles remain editable before the batch starts.

The interval is optional. With the interval disabled, every video assigned to a
day uses the exact same publication time. For example, 25 videos over 5 days at
21:00 produce 5 videos at 21:00 on each date. With the interval enabled, 3
videos starting at 21:00 with a 5-minute interval produce 21:00, 21:05, and
21:10. Up to 15 videos can be assigned to each day from the Web UI.

Before creation, subjects are normalized and compared semantically with the
current draft, previous manifests, task scripts, and upload history. Exact or
near-duplicate premises are rejected rather than silently replaced. Manifest
creation uses a cross-process lock so simultaneous batches cannot reserve the
same premise.

AI-prepared ideas use the `realistic_inspiring_balanced_v1` editorial profile.
Each brief must contain a protagonist, concrete problem, action, obstacle, and
believable result. A quality score blocks generic mystery templates and
unsupported claims before rendering. Script generation validates paragraph
count, duplicate paragraphs, invented numbers, and common sensational claims;
invalid responses are retried. Publishing metadata uses concise captions and a
balanced mix of concrete, emotional, curiosity, and strong-hook titles without
inventing stakes.

All requested subjects and publication slots are prepared before execution. The
default `interleaved` mode generates one video, schedules it immediately, and
then generates the next video. The optional `generate_all_first` mode renders
the complete batch before starting uploads.

Upload failures do not stop later videos from being generated. Transient and
quota failures remain in the manifest for automatic retry; uncertain outcomes
require review so a retry cannot create a duplicate upload.

Batch manifests are stored under `storage/youtube_batches`. They record every
subject, optional edited publishing title, idea mode, quality state, task ID,
generated file, publication slot, upload state, and error. Older manifests are
normalized in memory and remain compatible.
